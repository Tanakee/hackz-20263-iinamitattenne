# 🌊 いい波立ってんね～
### 議論を「物理現象」として体感する、新感覚の議論プラットフォーム

---

## 1. 課題意識

今のSNSでは、どんな意見も同じタイムラインに流れ、すぐに消費される。

「人類は進化を止めるべきだ」という壮大な問いかけも、「僕はラーメンが好きだ」というつぶやきも、同じ1投稿として扱われる。

**それっておかしくない？**

言葉には本来、重さがある。主語がデカいほど、大きな波紋を生むはずだ。

---

## 2. プロダクト概要

「いい波立ってんね～」は、あなたの意見を**石**として池に投じる議論プラットフォームです。

投稿すると、言葉は石になって池に飛んでいき、着水し、波紋を広げ、水底に沈んでいく。

**主語がデカいほど、石がデカくなる。**

自然言語処理エンジン（GiNZA）が文章を解析し、主語のスケールを0〜100で判定する。

<table>
  <thead>
    <tr><th>入力</th><th>主語</th><th>スケール</th></tr>
  </thead>
  <tbody>
    <tr><td>僕はラーメンが好きだ</td><td>僕</td><td>15</td></tr>
    <tr><td>女は海</td><td>女</td><td>70</td></tr>
    <tr><td>人類は進化を止めるべきだ</td><td>人類</td><td>100</td></tr>
  </tbody>
</table>

主語がデカい石は、大きな波紋を生み、重い着水音を響かせ、高い水柱を立てる。そして、なかなか風化しない。

---

## 3. なぜこの技術構成にしたか

このプロダクトの核心は「**言葉の重みを物理的に体感させる**」こと。それを実現するために、各技術を選んだ。

### Three.js + cannon-es（物理演算）

「石が沈む」「波紋が広がる」「石が積み重なる」——これらをただのアニメーションではなく、**リアルな物理演算**で動かしたかった。cannon-esによる剛体シミュレーションで、石は水底に沈み、先に投じられた石の上に本当に積み重なっていく。議論が積み上がる様子を、物理として体感できる。

### GiNZA（日本語NLP）

英語NLPと違い、日本語は「は」「が」の格助詞で主語が変わる。`nsubj`（主語）と`dislocated`（は-トピック）を区別し、「僕はラーメンが**好き**」の主語が「ラーメン」ではなく「僕」だと正しく判定するために、spaCy + GiNZAを選んだ。固有表現認識（GPE/LOC/ORG）も組み合わせ、地名・組織名も適切にスケーリングする。

### Web Audio API

着水音をスケールに応じてリアルタイム生成している。大きな石ほど低く重い音（50Hz）、小さな石ほど高い音（120Hz）。ノイズバーストと正弦波オシレーターを組み合わせ、ライブラリなしで水しぶきの音を合成した。

### WebXR（Pico 4 Ultra対応）

「石を投げる」という行為をVRで完結させることで、没入感を最大化した。コントローラーの振りの強さ・方向が飛距離と着水位置に直結する。VR空間内のスマホ型UIパネルからひらがなキーボードで文章を入力し、自分の手で池に投じる体験をVR内で完結させた。


### マイクロサービス構成（Docker Compose）

各責務を明確に分離した。

```
Frontend (Vue.js + Three.js + cannon-es + WebXR)
        ↓
Gravity API (Node.js/Express)  ←→  NLP API (FastAPI + GiNZA)
        ↓
Logic API (PHP)  ←→  MySQL
```

- **Gravity API（Node.js）**: 質量計算・AI要約・VRリモート同期
- **NLP API（FastAPI + GiNZA）**: 日本語主語解析・スケール判定
- **Logic API（PHP）**: 風化・熱量・DB管理
- **MySQL**: 投稿・インタラクション・風の永続化

## 5. 技術的チャレンジ・工夫した点

### 日本語主語解析の精度向上

日本語の「は」格（トピック）と「が」格（主語）の区別が最大の難所だった。GiNZAの依存関係ラベルで`dislocated`（は-トピック）と`nsubj`を区別し、`dislocated`がある文では`nsubj`を補語として除外するロジックを実装した。

```python
has_dislocated = any(t.dep_ == "dislocated" for t in sent)
if has_dislocated and token.dep_ == "nsubj":
    continue  # 「ラーメンが好き」のラーメンを除外
```

### 文字種による重力係数

漢字が多い文章（論文調）は重く、ひらがなが多い文章（口語調）は軽く。文字種の割合から重力係数（0.85〜1.25）を算出し、同じ文字数でも「重さ」が変わる。

### Web Audio APIによる着水音のリアルタイム合成

スケール値（0〜100）を正規化し、オシレーター周波数・ノイズ持続時間・ゲインをすべてスケールに連動させた。ライブラリゼロで水しぶきの音響を再現。

### VRリモート同期

スマホで文章を入力しながらVRで石を投げる体験のために、Gravity APIにインメモリの共有ステートを持たせ、スマホ↔VR間をポーリングで同期させた。

### 風化の連続値化

当初`weathered`はbooleanだったが、0.0〜1.0のFLOATに変更。経過時間に応じて石が徐々に透明になり、脈動しながら消えていく演出を実現した。

## 6. 今後の展望

- **リアルタイム同期（WebSocket）**: 複数ユーザーが同じ池を共有し、他者の石が飛んでくる体験
- **波紋の干渉**: 複数の石が生む波紋が物理的に干渉し合う演算
- **主語スケール辞書の拡充**: 現在約50語の辞書をコーパスベースで自動拡張
- **風化速度の個別化**: 主語スケールが大きいほど風化が遅くなるロジックの精緻化
- **常設展示**: ハッカソン会場のような公共空間に池を設置し、来場者が石を投じ続ける
